# 基金经理方法论矩阵 Skill

让 AI 能同时从 5 位中国顶级基金经理的视角分析任意基金或个股。

## 核心文件

所有方法文件在 `references/` 下：

| 文件 | 经理 | 风格标签 | 一句话框架 |
|------|------|----------|-----------|
| `method.md` | 郑希（易方达） | 景气度/科技成长 | 从全球视野出发，找通胀产业环节，选 ROE 低位有修复弹性、流动性够的标的 |
| `method_丘栋荣.md` | 丘栋荣（中庚） | 深度价值 | 基于不确定性定价的低估值价值投资，超额收益的唯一来源是买得足够便宜 |
| `method_张坤.md` | 张坤（易方达） | GARP/消费价值 | 自下而上选最好的生意（ROIC>10%+低杠杆+强议价），集中长期持有 |
| `method_刘格菘.md` | 刘格菘（广发） | 行业集中（科技制造） | 用价值的方法投成长，从供需格局入手做中观行业比较，选 4-5 行业非中性配置 |
| `method_周蔚文.md` | 周蔚文（中欧） | 均衡成长 | 基于点线面体分析，用 DCF 做 5-10-20 年长期定价，三种投资情形找最优解 |

辅助文件：
- `engine_design.md` — 多经理并排分析矩阵的设计文档
- `scorecard.md` — 郑希框架的 6 维度评分卡（100 分制）
- `method_template.md` — 如何蒸馏经理方法的模板
- `decision-map.md` — 构建历程的决策记录

## 用法：分析任意基金/个股

### 基金分析

1. 在 red-mantis 项目中运行数据准备：
   - `python <red-mantis>/scripts/score_fund.py <代码>` — 自动获取数据并打印证据档案
   - 或者 `python <red-mantis>/scripts/fetch_any_fund.py <代码>` — 只抓数据
2. 读各经理 method.md，对照该基金的持仓/行业/风格逐项评估
3. 输出并排对比矩阵

### 个股分析

个股需要财务/估值数据支撑。可用 AKShare 获取：
- `ak.stock_financial_abstract_ths(symbol, indicator)` — 财务摘要（ROE/净利润/营收）
- 没有脚本支持时，也可手动查找后填入评估

### 矩阵输出模板

```
┌─────────────┬───────────────┬───────────────┬───────────────┐
│             │ 郑希 (景气度) │ 丘栋荣(价值)   │ 张坤 (GARP)   │
├─────────────┼───────────────┼───────────────┼───────────────┤
│ 维度1       │ 分/满分 + 判   │ 分/满分 + 判   │ 分/满分 + 判   │
│ 维度2       │ ...           │ ...           │ ...           │
├─────────────┼───────────────┼───────────────┼───────────────┤
│ 一句话判语  │ # 契合/不契合  │ # 契合/不契合  │ # 契合/不契合   │
│             │   理由         │   理由         │   理由         │
├─────────────┼───────────────┼───────────────┼───────────────┤
│             │ 刘格菘(行业集中)│ 周蔚文(均衡)   │               │
├─────────────┼───────────────┼───────────────┤               │
│ 维度1       │ ...           │ ...           │               │
├─────────────┼───────────────┼───────────────┤               │
│ 一句话判语  │ # ...         │ # ...         │               │
└─────────────┴───────────────┴───────────────┴───────────────┘
```

**关键原则：**
- 各经理维度独立，不做维度统一/不做加权平均
- 判语用文字定性 > 数字定量
- 引用经理原话时标注自然出处
- 推演部分标注"按 XX 的方法推演"
- 无数据支撑的具体数字标注"需核实"

## 用法：讲解某经理的方法

直接读对应的 method_*.md 文件。每个方法每条都有原话佐证，引用时自然标注来源（如"他在 2024 年接受中国基金报采访时说"）。**不要把文件名、章节号写入回答。**

## 用法：推演

对于该经理没有直接表态的话题：
1. 读 method_*.md 掌握框架
2. **明确标注**"按 XX 的方法推演，不是他的原话"
3. 用框架逐条推演
4. 推演中需要核实的数字标"需核实"

## 引用与诚实

- 引用要忠于原文，不改写
- 分清三种话：**原话**（引出处） / **推演**（标注） / **需核实**（标清）
