## 架构概览

```
输入 (基金代码/个股代码)
    │
    ▼
┌─────────────────────────────────────┐
│       数据准备层 (data/)             │
│  fund: fetch_any_fund.py (已有)      │
│  stock: 财务/估值/行情 新数据源      │
└────────────┬────────────────────────┘
             │ 统一数据接口
             ▼
┌─────────────────────────────────────┐
│       分析层 (analysis/)             │
│                                      │
│  ┌── 郑希模块 ──┐  ┌── 丘栋荣模块 ──┐│

│  └──────────────┘  └──────────────┘│
│  ┌── 张坤模块 ──┐  ┌── 刘格菘模块 ──┐│

│  └──────────────┘  └──────────────┘│
```

## 1. 数据准备层

### 基金数据（已有，可直接复用）

`scripts/fetch_any_fund.py` 已提供：
- `净值业绩规模.json` — NAV 序列、规模、成立以来收益
- `季度持仓.json` — 逐季前十大重仓 + 持仓比例
- 从持仓可计算：集中度、换手率代理、行业分布、重仓股 ROE/PB 代理

### 个股数据（新）

需要的字段：

| 数据类别 | 字段 | 计算方式 |
|---------|------|---------|
| 财务 | ROE (近3-5年)、ROIC、毛利率、营收增速、净利润增速、FCF/净利润比 | 从财报取，可缓存 |
| 估值 | PE (TTM)、PB、PE 历史分位、PB 历史分位 | 从行情+财报取 |
| 行情 | 日均成交额、换手率、总市值 | 从行情取 |
| 行业 | 申万一级/二级分类 | 从东方财富取 |

**数据源建议**：AKShare（免费，无需 token，覆盖率好）

```
pip install akshare
```

关键接口：
- `ak.stock_financial_abstract_ths()` — 财务摘要（ROE/营收/利润）
- `ak.stock_individual_info_ask()` — 行情快照（PE/PB/总市值）
- `ak.stock_individual_spot_xq()` — 实时快照
- `ak.stock_zh_a_hist()` — 日线行情（用于历史百分位）

## 2. 分析层：各经理维度映射

每个经理模块接收统一数据接口，输出自己的维度评分 JSON。

### 2.1 郑希（已有）

沿用现有的 `scorecard.md` 6 维度（100 分制），已有 `score_fund.py` 做基础计算。保持不变。

| 维度 | 权重 | 数据来源 | 可自动 |
|------|------|---------|--------|
| 景气方向/通胀属性 | 25% | 持仓行业判断 | ❌ 需人/AI判 |
| ROE低位弹性 | 20% | 持仓个股 ROE 中位数 | △ 半自动 |
| 全球视野/中国比较优势 | 15% | 行业属性 | ❌ 需判 |
| 流动性 | 10% | 持仓个股市值+成交额 | ✅ 可算 |
| 集中度与周期拼接 | 15% | 前十大集中度+换手率 | ✅ 可算 |
| 业绩与回撤印证 | 15% | 净值/回撤 | ✅ 可算 |

### 2.2 丘栋荣

| 维度 | 数据来源 | 可自动 |
|------|---------|--------|
| 基本面低风险 | 负债率、FCF/净利润、行业格局 | △ 半自动 |
| 低估值（完整生命周期） | PB、PE历史分位、隐含回报估算 | ✅ 可算 |
| 可持续高增长/高弹性 | 营收增速、ROE改善趋势 | △ 半自动 |
| 供给格局（供要紧、需向新） | 行业产能周期判断 | ❌ 需判 |
| 是否处于"右偏分布" | 估值分位+盈利弹性 | △ 半自动 |

输出：三条件达标率 + 一句话判语。

### 2.3 张坤

| 维度 | 数据来源 | 可自动 |
|------|---------|--------|
| ROIC > 10%（5-10年） | 财务数据 | ✅ 可算 |
| 低杠杆 | 资产负债率 | ✅ 可算 |
| 强议价能力 | 毛利率、营运资本 | △ 半自动 |
| 自由现金流充沛 | FCF/净利润比 | ✅ 可算 |
| 安全边际（估值低） | PE/PB历史分位 | ✅ 可算 |
| 基础回报（股息+回购收益率） | 股息率+回购比例 | △ 半自动 |
| 期权价值（顺周期+科技） | 行业属性+竞争地位 | ❌ 需判 |

输出：ROIC 初筛通过率 + 安全边际评估 + 基础回报率 + 一句话判语。

### 2.4 刘格菘

| 维度 | 数据来源 | 可自动 |
|------|---------|--------|
| 供需格局有趋势性变化 | 行业周期判断 | ❌ 需判 |
| 产业趋势持续2年+ | 行业景气度判断 | ❌ 需判 |
| DCF四阶段（高速/中速/低速/永续） | 营收增速+行业阶段 | △ 半自动 |
| 全球比较优势 | 行业出口竞争力判断 | ❌ 需判 |
| 错误定价（超调机会） | 估值分位+近期跌幅 | ✅ 可算 |
| 利润非线性增长潜力 | 行业拐点判断 | ❌ 需判 |

输出：框架契合度（5分制）+ 最匹配的 DCF 阶段 + 一句话判语。

### 2.5 周蔚文

| 维度 | 数据来源 | 可自动 |
|------|---------|--------|
| 产业趋势低位+价值低估（情形一） | 估值分位+行业景气位置 | △ 半自动 |
| 产业趋势机会（情形二） | 行业趋势判断 | ❌ 需判 |
| 价值低估机会（情形三） | 估值分位+稳定性 | ✅ 可算 |
| 点线面体综合判断 | 多维数据 | ❌ 需判 |
| 均衡配置验证 | 行业分散度 | ✅ 可算 |
| 流动性风险 | 重仓股日均成交额 | ✅ 可算 |

输出：三种情形匹配度 + 行业分散度 + 一句话判语。

## 3. 输出层：并排对比矩阵

### 格式设计

```
┌─────────────────────────────────────────────────────────────┐
│  分析标的: 易方达信息产业混合 (001513)                      │
├─────────────┬───────────────┬───────────────┬───────────────┤
│             │ 郑希 (景气度)  │ 丘栋荣(深度价值)│ 张坤 (GARP)  │
├─────────────┼───────────────┼───────────────┼───────────────┤
│ 维度1       │ 25/25 ✓       │ 5/10 ✗        │ 8/10 ▲       │
│ 维度2       │ 18/20 ✓       │ 7/10 ✓        │ 6/10 △       │
│ 维度3       │ 12/15 ▲       │ 3/10 ✗        │ 4/10 ▲       │
│ 维度4       │ 8/10 ✓        │ ──            │ ──           │
│ ...         │ ...           │ ...           │ ...           │
├─────────────┼───────────────┼───────────────┼───────────────┤
│ 一句话判语   │ 风格高度契合   │ 不太会买       │ 质地不错但   │
│             │ 他就是这种打法 │ 估值不够便宜   │ 安全边际不够 │
├─────────────┼───────────────┼───────────────┼───────────────┤
│             │ 刘格菘(行业集中)│ 周蔚文(均衡)   │              │
├─────────────┼───────────────┼───────────────┤              │
│ 维度1       │ 20/25 ✓       │ 6/10 △        │              │
│ 维度2       │ 15/20 ▲       │ 7/10 ▲        │              │
│ ...         │ ...           │ ...           │              │
├─────────────┼───────────────┼───────────────┤              │
│ 一句话判语   │ 中观配置契合   │ 部分契合       │              │
│             │ 但基本面验证待查│ 需看底部区域   │              │
└─────────────┴───────────────┴───────────────┴───────────────┘
```

### 关键原则

1. **维度不统一** — 每个经理用自己的维度体系，不强行对齐
2. **不做加权总分** — 只输出各经理独立判断，让用户自己综合
3. **判语必须有证据链支撑** — 每个分数后面附带关键数据引用
4. **自动/半自动/人工标注清楚** — 表格中用 **✓ 自动 / △ 半自动 / ❌ 人工判断** 标记

## 4. 实现方案

### 方案 A：Python 脚本（推荐）

```
scripts/
├── analyze_fund.py         # 主入口：基金分析矩阵
├── analyze_stock.py        # 主入口：个股分析矩阵（依赖 akshare）
├── analysis/
│   ├── __init__.py
│   ├── base.py             # 分析器基类
│   ├── zhengxi.py          # 郑希模块
│   ├── qiudongrong.py      # 丘栋荣模块
│   ├── zhangkun.py         # 张坤模块
│   ├── liugesong.py        # 刘格菘模块
│   └── zhouweiwen.py       # 周蔚文模块
└── output/
    ├── matrix.py            # 矩阵渲染
    └── table.py             # 终端表格输出
```

### 方案 B：Skill.md 引导 + 人/AI 判

不写分析脚本，直接在 SKILL.md 中定义分析流程，由 AI 根据 method*.md 手动判分。

**推荐先用方案 B 验证矩阵价值，确认有用后再投入方案 A 的代码开发。**
