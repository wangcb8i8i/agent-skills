# 饱和 — 递归循环参考

综合完成后执行覆盖度评估。不满足关闭标准则补搜，直到饱和。

## 覆盖度评估

- 对照锚定点的每个维度评估：全面 / 部分 / 空白
- 识别置信度 < 70% 的核心主张
- 计算饱和度：本轮新增核心发现数 / 累计总核心发现数 × 100%

## 关闭标准

- **C1** — 所有决策关键维度至少「部分覆盖」（非关键维度允许空白）
- **C2** — 顶部 3 个决策因子置信度 ≥ 70%
- **C3** — 连续 2 轮未改变推荐结论（不是「没新信息」，是「新信息不推翻结论」）
- **C4** — 已达最大轮次上限（5 轮）

**通过 = C1 + C2 + C3 全部满足。** C4 是熔断机制——5 轮仍未收敛则强行关闭，报告标注未覆盖。

## 补搜策略

| 缺口类型 | 补搜方法 |
|----------|----------|
| 空白维度 | 放宽约束 + 同义扩展 + 上探一级概念 |
| 置信度 < 70% 的主张 | 搜索验证性或反驳性证据 |
| 未裁决的矛盾点 | 第三方信源（学术文献、官方数据） |
| 跨维度关联 | 一次覆盖多个缺口的综合搜索词 |

**执行：** 补搜 → 窄化搜索（仅限补搜项） → 增量合并到已有发现池，不覆盖原文 → 重新综合 → 回到覆盖度评估。
