# 报告结构参考

报告结构由问题本质驱动，不是固定模板。以下原则指导结构设计。

## 类型驱动主线

研究类型决定报告的「主干逻辑」：

| 研究类型 | 主线 |
|----------|------|
| 投资判断 | 多情景对比 → 风险矩阵 → 不对称机会 → 行动建议 |
| 技术趋势 | 现状与阶段 → 成熟度评估 → 关键信号 → 趋势判断 |
| 行业调研 | 产业链条 → 竞争格局 → 驱动/抑制因素 → 战略含义 |
| 产品决策 | 用户需求 → 竞品分析 → 选项对比 → 行动路线 |
| 生活经验 | 背景全景 → 核心选择 → 实操条件 → 成本收益 |
| 知识储备 | 领域地图 → 核心概念 → 脉络关系 → 进阶路径 |

## 角色决定视角侧重

用户角色影响哪些内容前置：

| 角色 | 前置侧重 |
|------|----------|
| 创业者/产品人 | 行动路线、可行性、资源条件 |
| 投资人 | 风险矩阵、回报率、竞争壁垒 |
| 研究者 | 方法论、数据完整性、信息缺口 |
| 技术决策者 | 技术对比、迁移路径、兼容性 |

## 问题粒度决定展开度

| 粒度 | 广度/深度配比 |
|------|--------------|
| 宽泛（如"SaaS 行业趋势"） | 全景概览 + 2-3 个深挖点 |
| 中等（如"中国 CRM 市场竞争格局"） | 维度展开充分 |
| 聚焦（如"HubSpot 和 Salesforce 在小企业市场的差异"） | 单点深挖，不留宽泛章节 |

## 决策类型驱动终端结构

收尾方式由决策类型决定：

| 决策类型 | 收尾结构 |
|----------|----------|
| 选型决策 | 前置对比框架 + 推荐 + 为什么不选其他选项 |
| 判断决策 | 前置阈值 + 证据 + 置信度 |
| 理解决策 | 全景 → 关键脉络 → 路标（下一步关注什么） |
| 监控决策 | 信号清单 + 阈值 + 响应策略 |

## 建议生成

每条建议包含：行动 + 预期效果 + 主要风险 + 前置条件。

按研究类型调整建议格式：

- 投资判断 → 行动建议（3-5 条，包含退出条件）
- 产品决策 → 行动建议（3-5 条，包含实施路线图）
- 生活经验 → 行动建议（3-5 条，附成本/收益）
- 行业调研 → 战略含义（对行业格局的影响，对不同参与者的差异化影响）
- 技术趋势 → 趋势判断与信号（关键先行指标 + 入场时机判断框架）
- 知识储备 → 知识体系梳理（从哪里开始、进阶路线、核心资源索引）

## 报告质量门

- [ ] 所有事实性主张有来源标注
- [ ] 结论有置信度标注
- [ ] 因果链有传递深度——不只是第一层解释
- [ ] 建议可执行——不是「要关注 XX」而是「做 XX，因为 YY，风险是 ZZ」
- [ ] 每条关键推荐标注了反转条件
- [ ] 不确定性章节区分了「值得查证」和「可以接受」
- [ ] 信息缺口已诚实标注
- [ ] 报告结构由问题本质驱动，而非固定模板
