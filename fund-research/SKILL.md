# 基金裁决

围绕一个裁决（买/卖/持有/对比）搜索证据、做出判断。输出一份用户不需要翻页就知道该怎么做的报告。

## 核心追问

每个分析维度只回答这三个问题：

> 1. 这个证据让你对判断改变多少？
> 2. 你有多确信？
> 3. 什么能翻案？

**重要**：这个框架只用于你的分析过程。报告里不出现"裁决""判决""呈堂证据"等词汇。

---

## 工作流

### 一、定焦

确认基金 + 案由，然后自动开始。用户不需要懂参数。

**1. 确定基金**
用户输入基金名称或代码。不完整时做模糊匹配引导确认。
→ 完成：单一基金已确认。对比案由 → 两到三只基金已确认。

**2. 确定案由**
案由决定研究重心。用户不知道选什么时，用一句话解释差异后建议一个。

- **买入** — 该不该买？什么时机合适？
- **持有** — 继续持有还是卖出？
- **监控** — 持续跟踪哪些信号？
- **对比** — A vs B（vs C），选哪个？

→ 完成：案由已确定。

**3. Scope 确认**
默认推理，不询问用户。用户如果明确给出则使用用户提供的。

| 参数 | 默认值 |
|------|--------|
| 时间跨度 | 近 1 年（除非历史数据有强信号需更长窗口）|
| 比较基准 | 同类均值（根据基金类型自动匹配）|

输出简短确认：「基金 + 案由 + 基准」核对，然后问"开始？"
→ 完成：用户确认「开始」或「是」。

**▸ 定焦出口检查**
- [ ] 案由已确定
- [ ] 基金已确认
- [ ] 用户只回答了一两个简单问题 — 没被要求指定技术参数

---

### 二、取证

全自动搜索、筛选、分析所有维度。中间不询问用户。

**原则**
- 每完成一个维度用 1 句非中断式更新进度
- 每个关键事实追问 Why 直到触及根因
- 所有事实性主张标注来源。无法确认的标「待验证」
- 每个维度同时回答：这个发现如果为真/为假，改变什么结论？

**0. 先验锚定**
在看具体基金数据前，加载该类基金的历史基率作为默认假设。

根据基金类型推理：
- 同类基金过去 3 年跑赢基准的比例（约 30%）
- 同类基金过去 5 年持续跑赢基准的比例（约 15%）
- 同类基金最大回撤中位数

基率是分析的起点 — 后续证据只在置信度足够时才应偏离这个锚。
→ 完成：基率已作为先验锚。

**1. 搜索**
优先级搜索专业数据源：天天基金 / 晨星 / Choice / 基金年报原文。
每维度至少 1 组定向搜索关键词。决策影响度高的维度优先。

信息不足时降级链：放宽时间 → 扩展同义词 → 上探一级话题。

> **工具约束**：WebSearch 可能无法直接获取部分专业数据（Brinson 归因、内部持仓穿透等）。
> 降级链：
> 1. WebSearch 基金名称 + 指标关键词
> 2. WebFetch 天天基金/晨星公开页面
> 3. 仍无法获取 → 标注「数据受限，无法分析」，不在报告中编造数据

→ 完成：每个高影响维度至少有 2 份可引用材料或已标注「信息有限」。整体材料 ≥ 5 篇，或已启动降级路径。

**2. 筛选**
每份材料二维评分：相关度（★★★/★★/★）和决策影响力（◉/◎/○）。
四维可信度评估：来源权威性 / 数据可验证性 / 时效性 / 潜在偏见。

**3. 全维分析**
加载 `references/evidence-rules.md` 获取各维度指标库和解读方法。

按决策影响度从高到低执行。每维输出：

```
维度名 | 核心指标 | 置信度 | 决策影响度 | 关键证据摘要 | 信息缺口
```

- 特别关注**「前瞻与情景」**维度：催化剂扫描 + 反方情景分析
- 对顶部 3 个最影响结论的主张执行反向搜索 + 压力测试
- 案由为**对比**：对每只基金独立执行本流程，然后额外构建差异矩阵

→ 完成：`evidence-rules.md` 中每个维度至少有一行证据或显式标注「数据不足无法分析」。不允许无声跳过任何一个维度。

**4. 证据汇总**
汇总所有维度，按「置信度 × 决策影响力」排序。识别 3 条最影响结论的关键证据。

**矛盾裁决** — 当不同维度指向矛盾结论时：
1. 检查矛盾是否可解释（高换手+高超额→交易型风格，非矛盾）
2. 不可解释 → 取置信度更高的维度
3. 置信度相近 → 偏向不利假设（最坏情况优先）
4. 仍无法解决 → 标注「核心矛盾未解决」，结论降一档可信度

核心维度置信度 < 50% → 输出「信息不足」报告，附翻盘所需信息。
→ 完成：3 条关键证据已识别。矛盾已裁决或已标注未解决。

**▸ 取证出口检查**
- [ ] 全部分析维度覆盖，未跳过
- [ ] 高影响维度置信度已显式标注
- [ ] 反向搜索 + 压力测试已对 top 3 主张执行
- [ ] 如证据不足 → 已输出「信息不足」报告而非伪造结论

---

### 三、输出

**原则**
- 第一条就是结论。用户不应该需要翻页才能知道答案
- 结论包含 **买/不买/持有/卖出/等信号** + 可信度 + 预期收益
- 用户只看到结论和证据，不看到分析方法论
- 翻案条件和信息缺口已标注
- 不出现"裁决""判决""呈堂证据"等 internal 词汇

**1. 撰写报告**
标题用「{基金名称} — {案由}研究简报」。

必须包含以下内容（顺序可调，可增加不属于模板的 section）：

| 类别 | 内容 |
|------|------|
| 通用 | 结论（买/不买/持有/卖出/等信号 + 可信度 + 预期收益）|
| 通用 | 关键证据（≥ 3 条，含置信度标注）|
| 通用 | 主要风险 + 翻案条件 |
| 通用 | 信息缺口 + 来源 |
| 买入 | 买入前提条件 + 退出信号 |
| 持有 | 卖出触发条件 |
| 监控 | 关键信号清单 + 建议行动 |
| 对比 | 对比矩阵 + 推荐理由 |

格式参考 `references/verdict-templates.md`，不受其限制。
→ 完成：结论以加粗单词开头。必须内容已覆盖。

**2. 输出**
首次输出直接显示在对话中。同时保存到磁盘。
输出路径：`docs/research/fund/{基金代码}-{案由}-{日期}.md`

**▸ 输出出口检查**
- [ ] 报告前 3 行直接给出结论
- [ ] 结论包含「买/不买/持有/卖出/等信号」+ 可信度 + 预期收益
- [ ] 没有「判决书」「裁决」「呈堂证据」等 internal 词汇
- [ ] 不讨论分析方法 — 只呈现结论和证据
- [ ] 翻案条件已标注
- [ ] 信息缺口已标注
