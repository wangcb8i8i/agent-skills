# 基金裁决

围绕一个裁决（买/卖/持有/对比），通过定量因子分析 + 定性证据验证，做出可审计的投资判断。输出一份用户不需要翻页就知道该怎么做的报告。

## 核心追问

每个分析维度只回答这三个问题：

> 1. 这个证据让你对判断改变多少？
> 2. 你有多确信？
> 3. 什么能翻案？

**重要**：这个框架只用于你的分析过程。报告里不出现"裁决""判决""呈堂证据"等词汇。

---

## 工作流

### 零、排除

在开始正式分析前，对不合格品一票否决。不满足红线指标的基金直接退出，不进入后续流程。

**排除项与红线：**

| 检查项 | 红线 | 否决理由 |
|--------|------|----------|
| 基金规模 | 主动 < 2亿 / 指数 < 1亿 | 清盘或流动性风险 |
| 经理从业年限 | < 3 年 | 历史业绩不足以判断能力 |
| 机构持有占比 | > 90% | 机构一旦撤离净值将剧烈波动 |
| 换手率 | > 500%/年 | 过度交易，隐含高摩擦成本+不稳定性 |

执行命令：
```
python scripts/run_analysis.py <基金代码> --case <案由>
```
脚本自动执行排除层检查。如未通过，打印否决原因并退出（exit code 2）。

**降级**：排除层数据依赖 AKShare。如某个检查项数据不可得，仅跳过该项，不阻断流程。

**手动跳过**：通过 `--no-exclude` 参数可跳过排除层，用于调试或二次分析。

→ 完成：基金已通过排除层检查，或已退出并告知用户原因。

### 一、定焦

确认基金 + 案由，然后自动开始。用户不需要懂参数。

**1. 确定基金**
用户输入基金名称或代码。不完整时做模糊匹配引导确认。
→ 完成：单一基金已确认。对比案由 → 两到三只基金已确认。

**2. 确定案由**
案由决定研究重心和因子权重。用户不知道选什么时，用一句话解释差异后建议一个。

- **买入** — 该不该买？什么时机合适？（权重侧重：收益 + 风险）
- **持有** — 继续持有还是卖出？（权重侧重：质量 + 稳定性）
- **监控** — 持续跟踪哪些信号？（权重侧重：资金行为 + 风格漂移）
- **对比** — A vs B（vs C），选哪个？（权重侧重：全维度相对排名）

→ 完成：案由已确定。

**3. Scope 确认**
默认推理，不询问用户。用户如果明确给出则使用用户提供的。

| 参数 | 默认值 |
|------|--------|
| 时间跨度 | 多窗口并行 1年/3年/5年 |
| 比较基准 | 同类均值（根据基金类型自动匹配）|

输出简短确认：「基金 + 案由 + 基准」核对，然后问"开始？"
→ 完成：用户确认「开始」或「是」。

**▸ 定焦出口检查**
- [ ] 案由已确定
- [ ] 基金已确认
- [ ] 用户只回答了一两个简单问题 — 没被要求指定技术参数

### 二、量证 — 定量因子分析

运行内置 Python 因子分析脚本，产出第一层输出（纯定量因子报告）。中间不询问用户。

**原则**
- 因子分析是客观计算层，不涉及买卖判断
- 所有指标标注计算窗口和同类分位

**0. 先验锚定**
在看具体因子数据前，加载该类基金的历史基率作为默认假设。

根据基金类型推理：
- 同类基金过去 3 年跑赢基准的比例（约 30%）
- 同类基金过去 5 年持续跑赢基准的比例（约 15%）
- 同类基金最大回撤中位数

基率是分析的起点 — 后续因子分析只在偏离足够大时才应视为信号。

**1. 运行因子脚本**
执行命令：
```
python scripts/run_analysis.py <基金代码> --case <案由>
```

脚本 exit code：`0`=正常完成，`1`=运行时错误，`2`=排除层未通过。

脚本自动执行：
- AKShare 数据获取（净值、基本信息、持仓、规模、经理信息、换手率等）
- **排除层检查**（不合格品 exit 2）
- 5 维度因子计算（1年/3年/5年并行，含卡玛比率）
- **风格锚定** — 因子暴露稳定性检测
- **Active Share** 计算
- 同类排名计算（按数据源原生分类）
- 稳定性 / 情景 / 敏感性检验
- **第一层评分**（收益/风险/风格/质量/费率，不含行为共识）
- **行为共识修正**（基于申购/赎回/机构动向等信号）
- 输出因子分析报告（Markdown + JSON）

**2. 读取因子报告**
脚本输出一份因子分析报告（JSON + Markdown），包含：

| 维度 | 因子指标 | 窗口 |
|------|----------|------|
| 收益 | 超额收益、信息比率、同类排名趋势 | 1yr/3yr/5yr |
| 风险 | 最大回撤、下行波动、VaR、回撤恢复时间、**卡玛比率** | 1yr/3yr/5yr |
| 风格锚定 | 因子暴露稳定性、行业集中度、行业漂移、**Active Share** | 3yr |
| 质量 | 业绩持续性、恢复时间 | 3yr/5yr |
| 费率效率 | 管理费率、费后超额、**费用效率比(Active Share÷费率)** | 当前值 |

每项因子值附带同类分位（前 25% / 中位 / 后 25%）和置信度标注。

**3. 因子分析摘要**
读取报告后，用一段话提炼核心发现：
- 该基金在哪些因子上有显著优势/劣势
- 同类排名整体水平
- 异常信号（稳定性预警、风格漂移等）

→ 完成：因子报告已生成，核心发现已提炼。如有脚本执行失败（网络/依赖问题），降级为 WebSearch 获取因子数据。

**▸ 量证出口检查**
- [ ] 因子脚本已运行（或已标注降级原因）
- [ ] 6 维度因子值和同类分位已获取
- [ ] 核心发现已提炼为 3-5 句话

### 三、取证 — 定向定性调查

以因子发现为线索，进行定向 WebSearch。中间不询问用户。

**原则**
- 搜索方向由因子分析驱动——因子异常值就是调查重点
- 每个关键事实追问 Why 直到触及根因
- 所有事实性主张标注来源。无法确认的标「待验证」

**1. 定向搜索**
以因子发现为线索，优先级搜索：

| 因子异常 | 追踪方向 |
|----------|----------|
| 超额收益异常高/低 | 是否是风格β？是否可持续？ |
| 风格漂移预警 | 经理是否有公开说明？持仓是否真的变了？ |
| 回撤异常 | 发生了什么事件？之后如何修复？ |
| 规模快速增长/下降 | 申购/赎回结构如何？机构动向？ |
| 经理任职期短 | 前任去向？新经理背景？ |
| 费率过高 | 同类对比？是否提供了匹配的 alpha？ |

搜索来源优先级：天天基金网 / 晨星 / Choice / 基金年报原文。
信息不足时降级链：放宽时间 → 扩展同义词 → 上探一级话题。

**2. 筛选与评分**
每份材料二维评分：相关度（★★★/★★/★）和决策影响力（◉/◎/○）。
四维可信度评估：来源权威性 / 数据可验证性 / 时效性 / 潜在偏见。

**3. 维度分析**
加载 `references/evidence-rules.md`，按决策影响度从高到低执行。每维输出：

```
维度名 | 核心指标 | 置信度 | 决策影响度 | 关键证据摘要 | 信息缺口
```

- 因子报告异常维度的追查优先执行
- 特别关注**「前瞻与情景」**维度：催化剂扫描 + 反方情景分析
- 对顶部 3 个最影响结论的主张执行反向搜索 + 压力测试

→ 完成：每个高影响维度至少有 2 份可引用材料或已标注「信息有限」。

**▸ 取证出口检查**
- [ ] 因子异常信号已追查
- [ ] 全部分析维度覆盖，未跳过
- [ ] 高影响维度置信度已显式标注
- [ ] 反向搜索 + 压力测试已对 top 3 主张执行
- [ ] 如证据不足 → 已标注

### 四、合成

将第一层（定量因子报告）与第二层（定性证据）合并，输出最终判断。

**1. 证据汇总**
列出所有证据（定量 + 定性），按「置信度 × 决策影响力」排序。识别 3 条最影响结论的关键证据。

**矛盾裁决** — 当定量与定性指向矛盾结论时：
1. 检查矛盾是否可解释（如高换手+高超额→交易型风格，非矛盾）
2. 不可解释 → 取置信度更高的证据
3. 置信度相近 → 偏向不利假设（最坏情况优先）
4. 仍无法解决 → 标注「核心矛盾未解决」，结论降一档可信度

**2. 第一层评分**
按案由驱动权重加权（不含行为共识）：

| 案由 | 收益 | 风险 | 风格 | 质量 | 费率 |
|------|------|------|------|------|------|
| 买入 | 35% | 25% | 10% | 20% | 10% |
| 持有 | 15% | 20% | 15% | 45% | 5% |
| 监控 | 10% | 25% | 25% | 25% | 15% |
| 对比 | 25% | 20% | 20% | 25% | 10% |

权重为推荐默认值，可根据证据实际置信度微调（需标注调整理由）。

**3. 行为共识修正**
在第一层评分基础上，根据资金行为信号做系数修正，决定"现在能不能买"。

| 信号 | 方向 | 修正幅度 | 适用案由 |
|------|------|----------|----------|
| 机构/内部人增持 | ↑ | +0.10 | 买入/对比 |
| 份额暴增（散户追涨） | ↓ | -0.10 | 买入/持有 |
| 封闭期打开高留存 | ↑ | +0.05 | 持有 |
| 大规模赎回 | ↓ | -0.10 | 买入/持有 |
| 历史恢复力已证明 | ↑ | +0.05 | 持有/监控 |

修正公式：`最终分 = 第一层分 × max(0.8, 1 + 案由缩放 × 修正系数和)`

| 案由 | 缩放系数 | 说明 |
|------|----------|------|
| 买入 | 1.0 | 行为共识充分影响决策 |
| 持有 | 0.5 | 只关注负面信号（赎回/追涨） |
| 监控 | 0.0 | 行为信号本身就是核心输出，不做修正 |
| 对比 | 0.7 | 相对比较，部分参考 |

**4. 输出最终判断**
- 判决：买 / 不买 / 持有 / 卖出 / 等信号
- 综合可信度：1-10 分
- 预期收益：区间（如 +5%~+15%）

可信度受以下因素影响：
- 因子稳定性（多窗口一致性高 → 可信度 +1~2）
- 定性与定量一致（一致 → 可信度 +1）
- 核心矛盾未解决 → 可信度 -2
- WebSearch 关键证据不可验证 → 可信度 -1

→ 完成：综合判决已输出，关键证据和可信度已标注。

**▸ 合成出口检查**
- [ ] Top 3 关键证据已识别（来自定量 + 定性）
- [ ] 矛盾已裁决或已标注未解决
- [ ] 案由对应权重已应用
- [ ] 最终判决 + 可信度 + 预期收益已确定

### 五、输出

**原则**
- 标题用「{基金名称} — {案由}研究简报」
- 第一层输出因子分析摘要（先呈证据）
- 第二层输出最终判断（再给结论）
- 结论包含 **买/不买/持有/卖出/等信号** + 可信度 + 预期收益
- 翻案条件和信息缺口已标注
- 不出现"裁决""判决""呈堂证据"等 internal 词汇

**报告结构（三层）：**

