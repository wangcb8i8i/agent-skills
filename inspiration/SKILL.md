# inspiration

**语言：所有产出物使用中文。**

从已理解的上下文中产生**灵感**——不是提取信息，是理解够了之后自然涌现的认知跳跃。灵感不是挖出来的，是长出来的。Agent 的工作是创造灵感涌现的条件，然后把它显性化。

## 工作方式

```
上下文（文档 / 代码 / recon / 对话 / 任意）
         │
         ▼
  ① 理解 —— 深入上下文，捕捉值得注意的点
         │
         ▼
  ② 蒸馏 —— 从"有意思"中筛出"能撑起追问"的
         │
         ▼
  ③ 发散 —— 以用户牵引方向为锚，追问"这意味着什么"
         │
         ▼
  ④ 捕捉 —— 把涌现的灵感显性化
         │
         ▼
  ⑤ 沉淀 —— 落盘到 .inspiration/inspirations.md
```

## 五个 Phase

### 1. 理解

深入理解用户提供的上下文。上下文可以是任意形态：文件路径、文档片段、代码、recon 产出、对话中已讨论过的事物。

用户未提供上下文时，检查当前工作目录下是否有 `.recon/`——有则自动作为上下文使用。没有则根据用户抛出的牵引方向，自行查阅项目文件、搜索相关代码、获取所需信息，建立理解。

理解的目标不是穷尽所有细节，而是捕捉到**值得产生灵感的东西**：反直觉的设计取舍、独特的实现路径、被刻意简化的地方、一个做法背后的约束链条。

**完成标准**：能用自己的话讲清上下文中值得产生灵感的东西——反直觉的设计取舍、独特的实现路径、被刻意简化的地方、一个做法背后的约束链条。不要求多少条，但每条必须是"这个点确实有点意思"。如果上下文本身太薄找不出任何值得注意的点——如实告知，不强行发散。

### 2. 蒸馏

从 Phase 1 的"值得注意的点"中分离信号与噪音。不是所有"有点意思"都能撑起有意义的追问——有些只能停在表面描述，有些能引出深层联想。

对每个点问：**从这里出发，能追问出什么？** 如果唯一的回答是"这个设计很有趣"，它过不了蒸馏。

**完成标准**：每个保留的点都能对应至少一个能追问的方向（提炼/迁移/反转/跃迁）。如果所有的点都撑不起追问——如实告知"素材不足以产生灵感"，结束。

### 3. 发散

以用户牵引方向为锚，对蒸馏后保留的点做系统化发散。四个追问方向驱动不同认知距离的探索：

| 方向 | 追问 | 认知距离 |
|------|------|---------|
| 提炼 | 这个做法可以抽象成什么模式？关键要素是什么？ | 近——从具体到抽象 |
| 迁移 | 这个模式搬到相邻领域，能解决什么？ | 中——跨领域平移 |
| 反转 | 如果约束反过来？他们刻意不做什么？这说明什么？ | 中——从负面空间找启发 |
| 跃迁 | 完全跳出这个领域，这个思路还能变成什么？ | 远——到达新品类 |

不要求四个方向都覆盖——在素材最肥沃的方向上深入。方向之间不是排他的，一条灵感可能跨越多个方向。

**有牵引时**：以用户指定的方向为主，其他方向为辅。

**无牵引时**：扫描所有值得注意的点，对每个点用四个方向试探，找出最肥沃的组合展开。

每一步跳跃都锚定在 Phase 1 的理解上——能指出灵感和源头的具体关联。

**完成标准**：每个被发散的方向都能说清"源头是什么 → 联想到了什么 → 为什么有意义"。空泛的"这个可以做 X"不算完成。素材确实不足以支撑任何有意义的发散时，如实报告，不编造。

### 4. 捕捉

把发散中涌现的灵感显性化。写之前确认两件事：核心想法能一句话说清、能追溯回上下文的具体位置。写完之后，如果有自然延伸的探索方向，顺手标注，但不强求。

**完成标准**：灵感已显性化。每条有清晰的"来源→想法"链条。

### 5. 沉淀

灵感落盘到当前工作目录的 `.inspiration/inspirations.md`。格式见 OUTPUT.md。

- 所有灵感写入同一个文件
- 每次追加在已有内容之后，不覆盖已有灵感
- 写完后不再复述文件内容

**完成标准**：灵感已追加写入 `inspirations.md`。写完后不再复述文件内容。

## 汇报规则

发现式汇报——汇报灵感本身，不汇报过程。

| 时机 | 汇报什么 |
|------|---------|
| 灵感产出时 | 灵感本身——想法是什么、从哪里来、可以往哪走 |
| 素材不足以支撑灵感时 | 如实告知缺了什么、为什么不足以产生有意义的联想 |

**禁止**："已完成理解""已发散 3 个方向""已捕捉 2 条灵感""文件已写入 .inspiration/"——这些是过程，不是发现。

汇报结束即止。不问"要不要继续深挖"——如果某个灵感确实有值得继续的方向，在捕捉时标注"还可挖"，由用户决定是否追问。

## 失败模式

| 问题 | 症状 |
|------|------|
| 理解不足就发散 | 灵感浮于表面，与上下文脱节。防御：Phase 1 完成标准——必须能说清值得产生灵感的东西，而非复述事实 |
| 发散变闲聊 | 联想越跑越远，失去与源头的锚定。防御：每条灵感必须能追溯到上下文中的具体依据 |
| 只有表层联想 | "这个可以做 X"但说不清 X 和源头的具体关联。防御：Phase 3 发散完成标准——必须说清"源头→想法→为什么有意义" |
| 素材贫瘠强行灵感 | 上下文本身不够丰富，强行产出"灵感"。防御：理解、蒸馏、发散三步都有"不足则如实告知"的出口 |
