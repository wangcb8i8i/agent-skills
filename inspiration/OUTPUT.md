# 灵感产出规范

> 被 [inspiration](SKILL.md) 引用的格式参考。只在你需要写文件或检查格式时查阅本节。

## 文件结构

当前工作目录下 **`.inspiration/`** 子目录：

```
{workspace}/
└── .inspiration/
    ├── INDEX.md          ← 灵感索引表（唯一入口）
    └── {slug}.md         ← 每条灵感一个文件
```

不写 `.gitignore`——灵感库默认进入版本控制。如果决定排除，自行添加。

## INDEX.md 格式

灵感索引表，每行一条灵感：

```markdown
# 灵感索引

| Slug | 标题 | 来源 | 日期 |
|------|------|------|------|
| llm-retry-pattern | LLM API 错误重试通用模式 | short-drama 项目 recon | 2026-07-05 |
| article-illustration-pipeline | 分镜管线搬到文章配图 | short-drama 项目 recon | 2026-07-05 |
| story-illustration-app | 故事图解应用 | short-drama 项目 recon | 2026-07-05 |
```

**字段规则**：

- **Slug**：灵感文件的 kebab-case 标识，与文件名一致（不含 `.md`）
- **标题**：灵感的一句话概括
- **来源**：上下文的简短标识（如项目名、文档名），不是文件路径
- **日期**：ISO 日期格式

按日期倒序排列，最新的在最上面。

## 灵感文件格式

每条灵感一个 `{slug}.md`：

```markdown
# {一句话标题}

> 来源：{上下文简短标识}
> 日期：YYYY-MM-DD

## 灵感

{核心想法——一段话讲清楚}

## 从哪里来

{上下文中的具体依据——不是引用文件路径，是描述上下文中的什么内容触发了这个灵感}

## 可探索方向（可选）

- {值得继续追问或深挖的方向}
- {另一个方向}
```

**注意**：灵感文件是给自己回看的，不是给别人读的报告。重点是记录想法和它的来源，不需要展开成文章。
