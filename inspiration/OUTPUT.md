# 灵感产出规范

> 被 [inspiration](SKILL.md) 引用的格式参考。只在你需要写文件或检查格式时查阅本节。

## 文件结构

当前工作目录下 `.docs/inspirations.md`，所有灵感归拢在一个文件：

```
{workspace}/
└── .docs/
    └── inspirations.md    ← 所有灵感
```

每次追加在文件末尾，不覆盖已有内容。

## inspirations.md 格式

```markdown
# 灵感

## {一句话标题}

> 来源：{上下文简短标识}

{自由叙述。可以有段落、列表、引用、表格——怎么表达清楚怎么来。
唯一约束：能看出灵感和源头的关联。}

---

## {下一条灵感的标题}
...
```

**规则**：

- 每条灵感以 `## {标题}` 开头
- 标题下用引用行标注来源和日期
- 内容自由，不强制内部小节
- 灵感之间用 `---` 分隔
- 来源用简短的上下文标识（如项目名、文档名），不是文件路径
- 追加到文件末尾，最新灵感在最下面

**注意**：灵感文件是给自己回看的。重点是记录想法和它的来源，不需要展开成文章。
