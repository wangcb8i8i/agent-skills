# 行业周期投资方法论

用行业周期视角分析板块、个股、基金的结构化分析框架。

三块内容：

- **方法** `references/method.md` — 完整的投资框架：找拐点、查通胀、选股、组合、退出。
- **评分卡** `references/scorecard.md` — 六维评分系统，衡量基金对此框架的契合度。
- **脚本** `scripts/` — 抓取真实基金数据、计算评分指标的辅助工具。

## 一句话框架

**从全球技术与行业拐点出发，沿产业链追踪定价权（通胀）集中的环节，落到本地有比较优势的位置，选流动性好、当前 ROE 低但有修复弹性的标的，在多周期之间轮动拼接，底层逻辑变坏时卖出。**

## 与主流方法的区别

| | 本方法 | 成长投资 | 价值投资 |
|---|---|---|---|
| 信仰 | 在**周期**，不在公司 | 在公司护城河 | 在安全边际 |
| ROE 偏好 | 低 ROE 待修复 | 高且稳定 ROE | 低 P/B、困境反转 |
| 换手 | 高（周期拼接） | 低到中 | 低 |
| 持有期 | 周期长度（季度~2-3年） | 多年 | 多年 |

## 使用方法

1. 读 `references/method.md` 了解完整框架。
2. 对基金评分时，读 `references/scorecard.md` 并先用 `scripts/score_fund.py` 备数据。
3. 分析时明确区分哪些是**框架推演**、哪些是**待核实的事实数据**。

## 边界

- 这是**分析框架**，不是预测模型。帮人结构化思考，不预测价格。
- 不构成投资建议。不推荐买卖。
- 评分卡衡量的是**对本框架的契合度**，不是基金绝对质量。一只基金可能优秀但评分低。
