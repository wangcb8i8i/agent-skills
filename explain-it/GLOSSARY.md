# GLOSSARY

## 回音

技能的目标。用户不仅能复述解释内容，还能预测未看过的代码行为。
完成后：用户说"哦，那如果 X 变了，Y 应该也会变"——这才是回音。
不是"记住答案"，是"能推演"。

## 缺口

用户理解中被识别的具体空洞。
每个缺口独立填补，填补后可能出现下一个。

## 刀锋

只读解决问题所必要的材料，不多读一行。
完成标志是可检查的：没有读过一行不需要的代码。
当发现一页文件只有 3 行有用，也只读那 3 行。

## 骨架

理解某个东西所需的最少结构。先出骨架再展开细节。
骨架应让用户能说出"这个系统的 3 块大砖是什么"。
骨架不是目录结构——是因果链。

## 地形

冷启动阶段的特殊骨架——3 个有因果关系的块：
1. 它解决什么
2. 它怎么解决（一句话）
3. 它不做什么（边界）

用户看完就知道整个系统在做什么、怎么来的、边界在哪。

## 回音序列

技能的主工作流：

```
入口 → 冷启动(可选) → 缺口分型 → 刀锋切入 → 骨架构建 → 填补循环
                                                          ↑
                                                    追问 → 重分型
```

冷启动只在入口处执行一次；之后的每次追问回到缺口分型，不回退到冷启动。

## 缺口簇

一个被表达的缺口周围存在一组未被表达的邻接缺口。
例如"这是什么"背后往往还藏着"由什么组成"（结构型）和"和什么不同"（对比型）。
技能的核心工作之一是在填补当前缺口的同时，探测缺口簇中尚未被覆盖的成员。
