# 理由型

用户想知道"为什么这样"——需要理解决策背后的权衡。

---

## 读取边界

读取范围：注释、docstring、决策记录（ADR）、设计文档的"理由"章节。

跳过：实现代码、配置、测试。

---

## 输出形状

```
决策情景 → 有哪些选项 → 决定性因素 + 被放弃的选项 → 当前方案
```

核心是要呈现**被放弃的选项**——没有 tradeoff 的"理由"不是理由，是借口。

---

## 视觉模式

### Decision table

```
| 决策 | 谁定的 | 理由 | 放弃的代价 | 状态 |
|------|--------|------|------------|------|
| ...  | ...    | ...  | ...        | ...  |
```

Tradeoff 列必须有值。如果没有明确 tradeoff 记录，标注`[未记录]`。

### Reasoning chain

论证结构可见时使用：

```
[前提 A] ──┐
            ├──▶ [推理] ──▶ [结论]
[前提 B] ──┘        │
                [假设: ...]
```

标注最薄弱的推理步骤。

### 选择规则

决策有记录 → Decision table。论证需要展开 → Reasoning chain。
都可以选 → Decision table（更浓缩）。

---

## 完成标志

用户能复述"为什么选了这个，放弃了什么"。
