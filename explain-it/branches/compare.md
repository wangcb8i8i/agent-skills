# 对比型

用户想知道"A 和 B 有什么区别"——需要聚焦决定性差异。

---

## 读取边界

读取范围：双方的定义和关键差异点。
按"决定差异"找——读两边的关键行为/签名/约束，不读共通的实现细节。

跳过：共同之处、各自的完整实现。

---

## 输出形状

```
对比表 → 决定性差异（不列共同点） → 一句话总结"什么时候用哪个"
```

如果共同点多于差异，先一句话点明它们本质相同，再列差异。
不要在表里浪费行写"都是 X"。

---

## 视觉模式

### Compact table

```
| 维度  | A        | B        |
|-------|----------|----------|
| 关键差异 1 | ...      | ...      |
| 关键差异 2 | ...      | ...      |
```

只含决定性差异。如果表超过 5 行，说明不是在对比——是在罗列。

### Options-compared（方案选型场景）

```
| 标准  | 选项 A  | 选项 B  | 选项 C（选中） |
|-------|---------|---------|---------------|
| ...   | ...     | ...     | ...           |
```

适用场景：多个方案选了其中一个。不适用于两个实例的差异对比。

### 选择规则

实例/概念对比 → Compact table。方案选型对比 → Options-compared。

---

## 完成标志

用户能说清"A 和 B 差在哪，什么时候该用哪个"。
