# 识别型

用户想知道"这是什么"——需要识别一个东西的身份和用途。

---

## 读取边界

读取范围：入口 / 标题 / 摘要。≤ 10 行。

跳过：所有实现细节、分支、配置、参数说明。

---

## 输出形状

```
痛点 → 它为了解决什么而生
是什么 → 一句话定义
不误用的关键事实 → 最容易弄错的一两件事
```

顺序固定：先说痛点再定义。跳过"是什么"直接给痛点也成立。

---

## 视觉模式

无图。只用文字。

有嵌套结构时，可用简略层级图辅助定位：

```
[某系统]
  └─ [当前组件] ← 你在这里
```

不要画调用图、不要画数据流、不要画状态机。

---

## 完成标志

用户能复述"它解决了什么" + "它是什么"。
