# Type-specific Natural Language Strategies

按配图类型选择对应的自然语言撰写策略。

本文件只回答一件事：当某张图已经在 PLANNING 阶段被确定为某种类型后，Prompt 应该如何侧重表达。

使用规则：

- 本文件服务于 `SKILL.md` 中的 Step 5 和 Step 6，不单独决定是否配图。
- 每种类型都必须服从 `Target Model Anchor`、`图内文字策略` 和 `信息溯源表`。
- Prompt 中的关键视觉元素必须能够回溯到三类来源之一：`原文依据`、`合理推导`、`视觉转化方案`。
- `禁止添加` 中列出的元素禁止出现在 Prompt 中。
- 当目标模型未指定时，默认采用模型无关、语义优先的自然语言描述。
- 当图中文字策略与某种类型的默认习惯冲突时，以图中文字策略为准。
- 需要选择抽象隐喻时，优先参考 `references/metaphor-guide.md`。
- 需要默认语义色板或稳定状态色时，优先参考 `references/tag-quick-reference.md`。
- 各节不再重复展开这两份 reference 的使用规则，默认适用上述约束。

## 概念示意图

适用条件：

- 用于解释抽象概念首次出现的段落。
- 用于展示关系网络、概念映射、机制说明。
- 适合“帮助理解”，不适合高精度数值表达。

Grounding 要求：

- 核心概念名词必须来自 `原文依据` 或原文术语。
- 拓扑结构、节点造型、关系隐喻可以来自 `合理推导`。
- 抽象概念转视觉元素时，优先使用 `视觉转化方案`，但不得引入原文未暗示的具体设备、人物或 UI。
- 如果原文没有出现明确数量，避免凭空指定过多离散实体。

实体描述：

- Primary 实体必须包含原文中的重要术语和名词。
- 可用拓扑、网络、能量流、容器等隐喻连接抽象与具象。
- 重点描述实体间差异，而不是堆积材质细节。

空间布局：

- 推荐径向、网络、节点-边、中心-外围布局。
- 以结构清晰度为第一优先级，省略非必要纹理和背景物。
- 当需要突出主概念时，使用明显层级中心。

色彩使用：

- 不同实体类型使用不同色相区分。
- 连接线颜色区分关系类型或状态变化。
- 同一概念跨图复用时，必须服从 Visual Glossary。

图中文字策略：

- 默认优先 `无文字` 或 `仅标签`。
- 当策略为 `仅标签` 时，只允许极少量结构标签，不写解释性句子。
- 当策略为 `仅数字` 时，仅在概念本身包含编号、阶段数或比例时使用。
- 当策略为 `必须精确文字` 时，仅保留最关键的短词，并逐字引用。
- 当策略为 `尽量无字` 时，优先通过颜色、发光、连接关系表达含义。

模型适配提示：

- 强调语义准确时，优先写清实体角色、关系方向和层级。
- 风格冲击优先时，可增强氛围、光照、材质，但不得掩盖结构关系。
- 排字能力弱的模型应避免要求节点内部有可读文字。
- 通用模型下优先写“形状 + 颜色 + 关系 + 布局”，少写抽象修辞。

Prompt 强调点：

- 先写主概念实体，再写次级实体和关系。
- 明确关系方向、连接方式、聚合或分散结构。
- 用视觉状态变化代替大段文字说明。
- 排除与原文无关的人物、品牌、真实设备界面和装饰性噪音。

## 架构图

适用条件：

- 用于表达系统边界、组件分层、服务关系、部署结构。
- 适合“帮助理解”，也适合轮廓图/概览图。
- 不适合主要依赖叙事情绪或隐喻氛围的场景。

Grounding 要求：

- 组件、层级、边界、连接方向优先来自 `原文依据`。
- 当原文未明确某条连接的协议、方向或层级时，不要伪造精确信息。
- 可用轻量 `合理推导` 补足空间布局，但不得改变系统关系。
- `视觉转化方案` 只用于增强层级可读性，不应用隐喻替代真实结构。

实体描述：

- 使用容器、盒子、层级板块、服务单元等隐喻。
- 每层组件应有清晰边界和角色区分。
- 如果有宿主层、网关层、存储层等，应自然体现层次归属。

空间布局：

- 推荐水平分层（L→R）或垂直分层（T→B）。
- 边界清晰，嵌套关系明确，跨层连接方向明确。
- 当系统较复杂时，优先减少透视感，保证示意清楚。

色彩使用：

- 每层不同色相或明度区分。
- 同层组件使用近似色，同类不同实例用饱和度变化区分。
- 边框和半透明底色可用于表达层级深度。

图中文字策略：

- 默认优先 `仅标签`。
- 当策略为 `无文字` 时，用结构分区和图例感构图替代标签表达。
- 当策略为 `仅数字` 时，仅保留节点编号、层号或端口编号等必要数字。
- 当策略为 `必须精确文字` 时，限制文字数量，只保留组件名、层名、协议名等短文本。
- 当策略为 `尽量无字` 时，优先生成无字结构底图，文字移到图外补充。

模型适配提示：

- 图表清晰优先时，减少材质、光影和景深，强化边界、方向和层级。
- 写实能力强的模型不代表适合架构图；不要引入真实机房、真实 UI 或照片感元素，除非原文明确要求。
- 排字能力弱的模型应尽量减少框内长标签。
- 通用模型下优先使用 `layered`、`containerized`、`bounded`、`directional links` 等结构词。

Prompt 强调点：

- 明确组件层级、边界、从属关系和连接方向。
- 强调“清晰分层”高于“视觉华丽”。
- 如果有入口、网关、存储、消息通道，应写清相对位置。
- 排除人物、摄影纹理、复杂场景背景和无关装饰。

## 流程图

适用条件：

- 用于表达时序、步骤推进、状态迁移、决策分支。
- 适合“流程说明”。
- 不适合仅有静态结构关系、没有明显时间顺序的内容。

Grounding 要求：

- 步骤顺序、分支条件、起止状态应优先来自 `原文依据`。
- 若原文未明确某个中间步骤，不要为追求完整而补造步骤。
- `合理推导` 只可用于布局和节奏，不应改变流程含义。
- `视觉转化方案` 可以增强状态变化，但不应替代流程节点本身。

实体描述：

- 步骤节点、决策节点、起止节点通过形状和颜色区分。
- 决策节点可用强调色，如 `#FF6B35`。
- 状态变化优先通过节点样式或箭头样式表达，而不是长文字解释。

空间布局：

- 推荐 L→R 或 T→B 时序布局。
- 多分支路径并行排列，避免箭头回绕过多。
- 泳道、阶段区、状态区可用不同底色区分。

色彩使用：

- 主流程与分支流程使用不同线型或不同色值。
- 成功、失败、等待、回滚等状态用稳定语义色表达。
- 同一状态跨图复用时保持一致。

图中文字策略：

- 默认优先 `仅标签`。
- 当策略为 `无文字` 时，用编号、箭头方向和状态色减少文字依赖。
- 当策略为 `仅数字` 时，可使用步骤编号、阶段号、百分比。
- 当策略为 `必须精确文字` 时，只保留关键动作词或状态词，逐字引用。
- 当策略为 `尽量无字` 时，流程节点尽量简化为符号化阶段块。

模型适配提示：

- 图表清晰优先时，减少环境描写，强化方向、节奏和分支结构。
- 排字能力弱的模型下，不要要求每个节点都出现长句。
- 对通用模型，优先写 `sequence`、`decision`、`branch`、`swimlane`、`directional arrows`。
- 风格冲击不是主目标，除非这是演示文稿用图。

Prompt 强调点：

- 先写起点，再写主流程，再写关键分支。
- 清楚说明箭头方向、分支条件、并行还是串行。
- 若存在异常路径或回退路径，要单独点明。
- 排除无关背景、人物场景和过度装饰。

## 信息图

适用条件：

- 用于数值、比例、分类、趋势、对比。
- 适合“数据对比”。
- 不适合没有清晰数据结构、只有抽象概念的段落。

Grounding 要求：

- 所有数值、比例、分类标签必须来自 `原文依据`。
- 如果原文没有数据，不要伪造柱状图、折线图、饼图数值。
- `合理推导` 仅用于选择图表类型和版式，不可改写数据关系。
- `视觉转化方案` 只作为辅助，不应干扰读者识别真实数据层级。

实体描述：

- 使用柱状图、折线图、饼图、雷达图、气泡图等图表隐喻。
- 主数据与次级数据层级要明确。
- 数值关系必须精确，不使用模糊描述替代原文数字。

空间布局：

- 推荐从上到下或从左到右渐进披露：标题区、概览数据、分类细节、图例。
- 对齐方式统一，视觉层级分明。
- 当数据较多时，优先减少装饰性背景。

色彩使用：

- 不同类别使用不同色相。
- 同一类别内部用明度或饱和度变化表达量级。
- 高对比色只用于强调关键数据点。

图中文字策略：

- 默认优先 `仅数字` 或 `仅标签`。
- 当策略为 `无文字` 时，只适用于非常简单的比例示意，不适合复杂信息图。
- 当策略为 `必须精确文字` 时，标题、分类名、数值必须逐字准确。
- 当目标模型排字弱时，优先生成无字图表骨架或仅数字版本，再把文本转移到图外。
- 不允许在未声明文字策略时要求整段图内说明文。

模型适配提示：

- 如果目标模型擅长排字，可适度保留短标题、数值和分类名。
- 如果目标模型排字弱，优先强调图表结构、留白和图例位置，不强求可读长文本。
- 通用模型下，尽量少用“复杂信息设计术语”，多用可视化结构描述。
- 图表清晰优先时，弱化情绪化光影和材质词。

Prompt 强调点：

- 明确数据类型、比较维度、图表结构和数值层级。
- 说明哪些是主数据、哪些是辅助数据。
- 写清标题区、图例区、图表主体的相对位置。
- 排除不必要装饰、摄影质感和会干扰识别的数据噪音。

## 装饰配图

适用条件：

- 用于封面图、社交分享卡片、章节开场图。
- 适合“装饰点缀”。
- 不适合承载高密度技术结构或精确数据。

Grounding 要求：

- 核心意象必须仍然能回溯到原文主题或主隐喻。
- 允许更高比例使用 `视觉转化方案`，但不得与原文事实冲突。
- 不应因追求氛围而加入错误的行业符号、错误时代感或错误场景。
- 若原文没有人物、品牌、设备，不要凭空加入会误导主题的具体对象。

实体描述：

- 聚焦主意象与整体气氛，而不是完整机制说明。
- 使用开放、松散、富有隐喻的视觉表述。
- 让一到两个核心意象承担主题，避免概念堆叠。

空间布局：

- 松散构图，不对称。
- 负空间建议 >=30%。
- 主题元素集中在一侧、中心或角落，给标题或后期留出空间。

色彩使用：

- 整体色调统一，不使用过多冲突色。
- 饱和度控制在合理范围，避免杂乱霓虹化。
- 若用于封面或社交卡，优先考虑远看辨识度。

图中文字策略：

- 默认优先 `无文字` 或 `尽量无字`。
- 当策略为 `必须精确文字` 时，只保留短标题或极少量关键短语。
- 若目标模型排字弱，建议固定为无字底图，标题转移到图外。
- 不要在装饰配图里塞入大量技术标签和说明文。

模型适配提示：

- 风格冲击优先时，可增加材质、光照、镜头、气氛词，但不能丢失主题可识别性。
- 社交卡片场景要特别关注横纵比和标题留白。
- 通用模型下，优先写主意象、光照、构图和留白，不要写过多微观结构。
- 如果目标模型偏写实，而需求实际上是抽象主视觉，应明确限制真实照片感。

Prompt 强调点：

- 先写核心意象，再写环境、构图和氛围。
- 强调负空间、远看辨识度和主次关系。
- 若服务于封面或分享卡，应预留文案空间。
- 排除无关技术标注、密集标签、真实 UI 和杂乱背景。
