# breakout

**语言：所有产出物使用中文。**
**leading word：破局。**

Breakout 是一种从"没想法"到"有方向"的快速发散框架。你不是缺乏判断力，而是眼前只有一条路（或根本没有路），需要有人帮你破局——把其他可能的路摆到桌面上。

## 触发

手动触发。在对话中输入 `#breakout` 后接你的模糊困扰。

```
#breakout 这个页面设计不太对劲
#breakout 支付转化率上不去，不知道加什么功能
#breakout 想做个新方向，但想不到做什么
```

## 适用场景

breakout 只在以下场景使用。不在这些场景时，不要自动触发。

### 你能感受到问题，但说不清楚
- "这个页面设计太丑了" → 到底是哪里丑？
- "这个产品体验不好" → 到底哪里不好？
- "感觉不对" → 该往哪调？

### 你知道要做事，但不知道做什么
- "想加个功能但没想法"
- "要改进一下，没有方向"

### 你只有一个选项，想知道还能怎么走
- "方案 A 还行但总觉得不够好"
- "只想到这么干，但不甘心"

## 核心流程

破局分为三个阶段。每阶段完成才进入下一阶段。

### 阶段一：发散

用户给一个模糊不满 → 产出 **3-4 个方向**。

**方向的标准格式：**
- **方向名**（2-4 字，易记易区分）
- **一句话展开**（为什么这个方向能破局，尽量用具体例子）

```
A. 信息聚焦 —— 所有元素平铺是平庸的根源。只突出前 3 个信息，其余弱化。
B. 留白呼吸 —— 间距拉大 2 倍，颜色字体减半。太挤太杂是问题所在。
C. 减法极致 —— 砍掉 80% 内容，只留一个信息和按钮。平庸因为装太多。
```

**保证方向真的不同**：出完方向后回头看一眼——它们出发的逻辑是不是同一个？如果是，重来。

**勘察范围**：以当前上下文为主。必要时（判断当前素材不足以支撑高质量方向时），主动联网搜索参考案例。从结果质量出发，不设限。

**完成标准**：用户选了一个方向。**不准擅自替用户选**。

### 阶段二：展开

对用户选定的方向 → 产出 **3-5 个具体动作**。

**动作的格式：**
- **动作名**
- **怎么做**（一两句，要具体到可执行）

```
B1. 放大间距 —— 主要模块间距从 20px 拉到 48px，让每个模块独立成区。
B2. 减色去噪 —— 只保留主色 + 中性灰，去掉辅助色和背景色块。
B3. 字体管控 —— 限到 1-2 种字体，用 weight 和 size 做层级。
```

**完成标准**：用户选了一个或多个动作，或要求继续进入阶段三。多选时阶段三将整合为一个提案。

### 阶段三：收敛

将用户选定的动作整合为完整提案——说清思路、理由、预期效果，形成闭环。用户多选时融合为一个提案。

**完成标准**：用户认可最终产出。产出形式取决于场景——可以是提案文档、设计决策记录、思路地图等。

## 多轮机制

第一轮发散用户不满意 → **不直接重发一轮**。流程是：

1. 用户给反馈（"太偏了""不够新""换一个角度"）
2. 根据反馈调整方向，再发一轮（3-4 个新方向）
3. 用户仍不满意 → 如实告知：当前场景暂时给不出有价值的破局方向

## 退出条件

两种终端状态之一：

1. **成功产出** — 经过发散→展开→收敛，产出闭环提案
2. **如实告知** — 多轮发散后仍无法找到有价值的破局方向
