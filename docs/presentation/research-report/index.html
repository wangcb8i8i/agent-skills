Research Report

自动化深度研究引擎

把「一个问题」变成「一份可拍板的结论」——不只是描述事实，而是穿透到因果层、支撑具体决策。

版本 2.1.0 委托式研究模式 因果链驱动分析

Research Report 是一个自动化的深度研究引擎。它的核心哲学是委托模式：你定方向，我做研究，你拿结果。Scope 确认后全程自动执行，仅在起点和终点需要用户参与。它不追求信息堆砌，而是用因果链追问穿透到结构动因，产出可直接用于决策的高质量报告。

与传统 AI 研究协作不同——后者需要用户全程介入、反复确认、不断引导——Research Report 把研究从「需要全程陪伴的协作」变成了「可以全权托付的委托」。中间过程完全自主，遇到障碍自主降级处理，最终交付的是一份有来源追溯、有置信度标注、有因果深度的结论。

第一章

为什么需要改变

传统 AI 研究协作模式存在一个根本性问题：用户需要全程介入。每一次中间确认都打断思路；每一轮补充引导都丢失前文语境；每多一次交互，决策周期就延长一天。

更深层的问题是：用户真正想要的不是有人陪着一起查资料，而是可以直接拿去拍板的结论。但现有模式受限于「协作式」设计——每步都需要人确认，看似安全，实则低效。

传统模式

协作式——用户全程参与每一步。搜索要确认方向，分析要确认框架，发现要确认重要性。结果是报告保质期长，但交付周期更长。

委托模式

委托式——用户只在起点定方向和终点拿结果。中间完全自主决策，包括遇到信息缺口时自主降级处理。交付质量和速度因此解耦。

第二章

SCOPE 阶段：唯一需要你的地方

Scope 是唯一需要用户深度参与的阶段，包含五个步骤：

问题澄清
把用户的兴趣短语转化为可回答的精确问题。用户的输入就是起点，不做强行追问——根据输入的具体程度做适应性引导。
锚定点确认
锚定点是约束研究方向的唯一机制。包括时间范围、地域范围、排除项。锚定点应该具体到能过滤信息，而非泛泛的方向。
已有知识注入（可选）
用户可在此提供已掌握的信息或材料，避免重复已知。如提供了原始材料，则首先扫描这些材料定位相关段落。
格式选择
Full Report（完整报告）、Executive Brief（一页简报）、Slide Deck（逐页要点）、Mind Map（思维导图）。格式只影响呈现方式，不影响研究深度。
Scope 确认
输出研究任务确认书，明确研究问题、类型、锚定点、范围、格式。确认后进入全自动模式。
第三章

AUTO-RESEARCH：自动执行

用户确认 Scope 后，研究进入全自动执行阶段，无需任何中间确认。

中间不询问用户意见。遇到缺口、矛盾、不确定——自主决策并记录，在报告中呈现。

搜索

生成多组搜索关键词，确保每个锚定维度至少被一组覆盖。信源多元化：行业报告 + 一手信息 + 新闻报道 + 社区讨论 + 学术文献。至少 5 篇以上优质材料。

筛选

每份材料做相关性分级（★★★/★★/★）和四维可信度评估：来源权威性、数据可验证性、时效性、潜在偏见。锚定点作为加权条件。

分析

分析框架由研究类型决定，不是套用模板。因果链追问贯穿始终——每个关键发现追问 Why 直到触及结构动因或底层约束。

综合

汇总所有角度发现，标注来源和置信度。交叉验证，矛盾裁决。按「置信度 × 对决策的影响力」排序。生成可执行建议。

第四章

分析视角工具箱

Lens Catalog 是一组分析视角的参考工具箱。关键是不在于用了几个 Lens，而在于是否覆盖了核心问题的所有关键维度。当议题需要独特的拆解方式时，可以自由创建新的 Lens。

/DEEP

深度因果挖掘

穿透表象，理解本质。三层追问：What（事实）→ How（机制）→ Why（动因）。递归追问直到触及底层约束。

/ANGLE

多角度对比

通过对比暴露差异——竞品之间、不同地区、不同时期。差异背后的原因分析驱动关键洞察。

/HYP

假设推演

不满足于描述现状，推演可能的未来。每个假设标注触发条件、概率评估和影响程度。

/CHALLENGE

挑战与反面证据

主动寻找反面证据，对抗确认偏误。对每个强主张做否定式搜索和压力测试。

/ACTION

行动转化

把分析变成决策。每项行动标注约束条件（资源/时间/能力）和成功标准。

/VOICES

利益相关方映射

理解不同参与者的立场、动机和信息差。各方之间的利益一致与冲突关系。

第五章

因果链追问

这是 Research Report 最核心的分析原则。每个关键发现都不只是记录事实，而是追问 Why——递归追问，直到回答触及无法继续分解的底层动因或结构约束。

因果链示例：市场下滑

市场下滑（事实）→ 因为头部客户流失（近因）→ 因为产品代际差距（中游）→ 因为研发投入连续 3 年低于行业均值（根因）

层层追问，不止于第一层解释

关键技巧是区分近因和根因。报告呈现必须至少传递两层深度。不是做一次「因为…」就停，而是递归追问。同时主动寻找反面证据，避免单一视角的确认偏误。

第六章

报告输出：由问题驱动

报告结构不是固定模板——由研究类型、用户角色和问题粒度共同决定叙事方式。不同研究类型的报告应该有明显的结构差异。

研究类型 报告主线
投资判断多情景对比 → 风险矩阵 → 不对称机会 → 行动建议
技术趋势现状与阶段 → 成熟度评估 → 关键信号 → 趋势判断
行业调研产业链条 → 竞争格局 → 驱动/抑制因素 → 战略含义
产品决策用户需求 → 竞品分析 → 选项对比 → 行动路线
知识储备领域地图 → 核心概念 → 脉络关系 → 进阶路径
第七章

异常降级：当信息不足时

遇到信息障碍时不卡住、不硬推、不伪造。诚实呈现信息状态比假装确定更有价值。这是 Research Report 在处理真实世界不确定性时的核心纪律。

搜索无结果

放宽搜索约束 → 切换信源类型 → 类比外推（标注「非直接证据」）→ 最终标注「信息缺口」，跳过该维度。不强求给出判断。

来源矛盾

定位矛盾点 → 来源可信度加权对比 → 检查时间差 → 无法裁决则保留双方立场，降级置信度。不强行统一。

来源质量偏低

所有低质量源结论标注「低置信度」→ 三角定位：用多源交叉印证 → 报告诚实呈现信息局限性。不假装确定。

语言壁垒

切换搜索语言重试 → 仍无法获取则标注「语言壁垒」。不影响已有语言的结论质量。

第八章

质量门禁与运作原则

Research Report 通过五道质量门禁确保产出质量，并由七项运作原则指导日常执行。

五道质量门禁

✔ 所有事实性主张有来源标注
✔ 结论有置信度标注
✔ 因果链有传递深度
✔ 建议可执行——不只是「要关注 XX」
✔ 信息缺口已诚实标注

七项运作原则

委托模式 · 来源必追溯 · 不确定就说
因果链必追问 · 结构由问题驱动
输出语言匹配 · 缺省不补搜

适用场景

投资判断

市场竞争格局、投资标的评估、行业前景判断——需要多情景对比和风险矩阵时

技术趋势

新技术成熟度评估、演化阶段判断、替代可能性分析——需要穿透技术 hype 时

产品决策

竞品差异分析、用户需求验证、路线图优先级——需要在信息不完备时做决策时

行业调研

产业结构、竞争格局、驱动力分析——需要快速建立行业认知时

Executive Summary

把「一个问题」变成一份可拍板的结论

不只是描述事实——穿透到因果层、支撑具体决策。全程无需用户介入中间过程。不是协作式研究助手——是委托式研究引擎。

Why Change

传统协作模式的困境

协作式（传统）

用户全程介入

每步都需要确认——搜索方向、分析框架、发现重要性。思路不断被打断。

上下文不断丢失

每次交互都可能丢失前文语境，导致重复工作和理解偏差。

决策周期长

从提问到结论的路径被中间确认节点拉长。一个简单的调研也可能拖几天。

委托式（Research Report）

只在起点和终点出现

用户在 Scope 阶段定方向，研究交付时看结论。中间完全自主。

自主决策与降级

遇到信息缺口、来源矛盾、质量偏低——自主处理，在报告中诚实标注。

一次交付，完整结论

交付的是一份有来源追溯、有置信度标注、有行动建议的完整报告。

Phase 2

AUTO-RESEARCH — 四个阶段

全自动执行，无需用户介入。锚定点作为硬约束贯穿全过程。

搜索

多元信源覆盖

多组搜索关键词，确保每个锚定维度被覆盖。至少 5 篇优质材料，覆盖 3+ 类信源。主动检查信息缺口。

分析

研究类型驱动框架

从问题本质推导拆解方式。因果链追问贯穿所有角度——区分近因和根因。

筛选

质量优于数量

相关性分级 + 四维可信度评估（权威性/可验证性/时效性/偏见）。标注来源矛盾并留到分析阶段处理。

综合

按决策价值排序

交叉验证、矛盾裁决。按「置信度 × 决策影响力」排序。生成 3-8 条可执行建议，每条标注回应了哪个锚定约束。

Phase 3

报告输出：结构由问题驱动

不是固定模板——报告结构由研究类型、用户角色和问题粒度共同决定。

类型驱动主线

投资→情景对比/风险矩阵；技术→成熟度/关键信号；行业→产业链/竞争格局

角色决定视角

投资人→风险/回报；产品人→行动路线/可行性；研究者→方法论/信息完整性

粒度决定展开度

宽泛问题→全景+2-3深挖点；聚焦问题→单点深挖不留宽泛章节

标准组件（但不强制）

摘要（必有）+ 正文 + 行动建议/战略含义 + 不确定性 + 来源
