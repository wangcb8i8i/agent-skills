# explain-concept

用户抛出一个概念，就是在**概念空间**里设了一个**坐标原点**。解释 = 围绕原点沿多个维度**探索航行**，在每个响应结束时自然地**收束**。

术语（原点、版图、迷雾、航行、收束等）是内部导航隐喻，只在技能内部使用。对外输出时用自然语言而非这些术语——不说「定原点」，直接确认概念；不说「收束」，直接总结。

## 原则

1. **意图决定半径**——同样问"闭包"，在读代码遇到障碍和系统学 JS 需要的探索半径不同。先判意图，再定航程。
2. **在版图内航行**——优先用已知概念做参照坐标。进入迷雾是成本，控制它。
3. **够用即停**——探索到用户能在版图内定位原点即可，不必遍历所有维度。
4. **版图累增**——版图在每次响应中累增，不重新扫描全领域。

## 流程

每次响应按以下阶段运行，从用户的 query 出发，到输出 + 归档结束。

### 1. 定锚

从用户的 query 中识别指向的概念和意图：

| 用户行为 | 指向 | 处理 |
|---------|------|------|
| 第一次问一个概念 | 新概念 | 设为原点 |
| 追问当前概念的某个维度 | 同一概念，新深度 | 沿用已绘版图，沿该维度继续 |
| 问另一个概念 | 版图内节点或空间外概念 | 根据是否在已有版图中，设为参照点或新原点 |
| 对比两个概念 | 关系探索 | 两者都纳入版图，边界测绘优先 |

**定原点：** 确认用户指的是哪个概念空间（同一个词跨域时）。工作空间也是版图的一部分——项目类型、技术栈、当前文件都是推断概念空间的锚点。

**测半径：** 用信号推断意图，决定探索范围：

| 信号 | 意图 | 探索策略 |
|------|------|---------|
| 前文讨论实现时突然问 | 遇到障碍 | 小半径：本质+边界够用就停 |
| 主动对比两个概念 | 校正混淆 | 中半径：边界测绘优先 |
| "我在学 X" | 系统学习 | 大半径：沿多个维度均匀展开 |
| "X 和 Y 选哪个" | 做决策 | 中半径：关联优先，弱化位置 |
| "你刚提到 X" | 好奇心延伸 | 小半径：本质+关联 |
| 无上下文直接问 | 未知 | 先反问定半径 |

拿不准 → 小半径起步，用户反馈后再扩展。

**完成标志：** 目标概念已识别（跨域歧义已消歧），意图对应的探索策略已确定。

### 2. 绘版图

先确认海图：目标概念在概念空间中的位置是否清楚（属于哪个领域、和什么概念相邻）。模糊 → 查资料（WebSearch、文档、代码）补到能画大致版图再继续。完全查不到 → 诚实标注位置未知，不硬画。

从已积累版图中找已知坐标（对话历史 + `.docs/` 归档文件中已有的概念直接作为已知坐标纳入）。用户主动提过的概念是版图上的锚点，没提过的当未知区域。

扫领域：展开目标概念在概念空间里的自然邻居圈——相邻、同级、配套、相反。这是领域知识扫描，先不管用户知不知道。

交叉过滤：
- 在版图内的 → 参照坐标
- 不在版图但强关联 → "借来定位"候选，标记迷雾
- 无关的 → 丢弃

选参照坐标：
- 优先从版图内已知点中选
- 如需引入新术语 → 标记为"借来定位，不用记"；或另选一个已知点
- 参照坐标够定位即可，过多会淹没原点

**完成标志：** 至少有一个已知点做参照坐标，可画出原点在版图上的粗略位置。

### 3. 展开

沿维度从原点向外探索。按概念类型和意图，从维度池中选最相关者展开：

| 维度 | 探索什么 | 何时选 |
|------|---------|--------|
| **本质** | 它是什么，一句话定义 | 总是 |
| **动机** | 为什么存在，解决什么问题 | 总是，一两句即可 |
| **边界** | 它不是什么，与已知点的差异 | 有易混淆邻居时 |
| **位置** | 上层归哪类，下层有什么子类 | 概念属分类体系时 |
| **关联** | 和谁配套、同场景、协作 | 需知道怎么用时 |
| **机制** | 内部如何运作 | 技术/过程类概念 |

每个维度探索完，用白话总结该维度发现。

**空间关系的呈现：** 位置、边界、关联三个维度涉及概念间的空间关系。展开时优先用可视化辅助——
- 层级树（缩进列表或 mermaid 简图）表示**位置**的归属与分类
- 对照表或维恩边界描述表示**边界**的差异
- 关系网（配套、协作、上下游）表示**关联**

本质、动机、机制三个维度不需要可视化。

探索中遇到陌生术语时：

| 角色 | 识别信号 | 处理 | 深度规则 |
|------|---------|------|---------|
| 参照坐标 | 去掉它，目标概念的表述仍然完整 | 一句话定义 + 标记"借来定位" | 不展开 |
| 必要前置 | 去掉它，目标概念的表述会断裂 | 简短解释后回到原概念 | 能连接回目标就停，不追溯前前置 |

**中止条件（满足任一）：**
- 已覆盖意图所需半径
- 下一维会进入**迷雾**区域且无替代路径
- 用户反馈已理解

### 4. 收束

每次响应结束时自动做两件事：

**总结：** 用白话归纳本次覆盖的概念本质、位置和关键关联。不依赖导航术语，不说「定原点」「收束」等。

**归档到 `.docs/`：** 将本次覆盖的维度写入当前工作目录的 `.docs/explain-<topic>.md`。

**文件定位：**
- 根据概念所属领域推断 topic 名（如 `js-runtime`、`database-indexing`）
- 目录不存在则创建

**文件结构：**

```
# <Topic 名>
> 概念空间简介

## <概念 A>
> 一句话本质

### 位置
### 边界
### 关联
### 动机
### 机制

---

## <概念 B>
...
```

**追加规则：**
- 文件中尚无该概念 → 追加新概念单元
- 文件已有该概念 → 覆盖已有维度，追加新维度
- 只写入本次实际覆盖的维度，未展开的不留空节
- 可视化内容（mermaid、缩进树等）原样写入对应小节
- 不同概念不合并

**完成标志：** 总结已输出，归档已写入。
